## Key Differences from GCS-Based Workflow

| Feature | GCS-Based | Repo-Based |
|---------|-----------|------------|
| **Script Source** | GCS Bucket | Git Repository |
| **Script Download** | `gsutil cp` from bucket | Cloud Build clones repo |
| **Result Storage** | GCS bucket file | Cloud Build output |
| **Setup Required** | GCS bucket with scripts | Git repository with validation folder |

## Prerequisites

1. **Git Repository** with validation scripts in a folder (e.g., `validation/`)
2. **Cloud Source Repositories** connected OR GitHub/Bitbucket repository
3. **Service Account Permissions** (same as GCS-based workflow)
4. **Cloud Build** must have access to clone the repository

## Configuration Parameters

### Required Parameters

| Parameter | Description | Example |
|-----------|-------------|---------|
| `project_id` | Source project where candidate image exists | `sandbox-dev-478813` |
| `image_id` | Name of the image to validate | `test-rhel-vm-image` |
| `display_project_id` | Target project for image promotion | `spoke-project1-476804` |

### Optional Parameters

| Parameter | Default | Description |
|-----------|---------|-------------|
| `repo_name` | `gcp-ansible-playbooks` | Name of the Git repository |
| `repo_branch` | `master` | Branch to clone from |
| `repo_dir` | `validation` | Directory containing validation scripts |
| `service_account` | (empty - uses default) | Custom service account for VM |
| `zone` | `us-central1-a` | Zone for VM creation |
| `machine_type` | `n1-standard-1` | Machine type for validation VM |

## Workflow Steps

### Step 1: Repository Cloning

```
Cloud Build automatically clones the repository:
  - Repository: {repo_name}
  - Branch: {repo_branch}
  - Directory: {repo_dir} (e.g., "validation")
  
Result: Validation scripts available in Cloud Build workspace
```

### Step 2: VM Creation

```
Creates validation VM:
  - Image: {image_id} from {project_id}
  - Zone: {zone}
  - Machine Type: {machine_type}
  - OS Login: Enabled
  - External IP: Enabled (for SSH access)
```

### Step 3: Script Deployment

```
Copies validation scripts from Cloud Build workspace to VM:
  - Source: Cloned repo files in Cloud Build
  - Destination: ~/validation/ on VM
  - Method: SCP or SSH with direct copy
```

### Step 4: Validation Execution

```
Runs validation on VM:
  - Script: validate_all.sh
  - Location: ~/validation/
  - Captures exit code (0 = Pass, non-zero = Fail)
```

### Step 5: Result Storage

```
Stores result in Cloud Build output:
  - Format: JSON
  - Location: $BUILDER_OUTPUT/output
  - Workflow reads from build output (not GCS)
```

### Step 6: Promotion (if Pass)

```
If validation_result == "Pass":
  - Copy image from {project_id} to {display_project_id}
  - Add labels: source_image, source_project, validation_status, promoted_date
  - Wait for image to be READY
```

### Step 7: Cleanup

```
Deletes validation VM:
  - Stop VM
  - Delete VM
  - Note: Candidate image is preserved
```
